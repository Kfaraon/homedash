**HomeDash** — это минималистичная веб-панель для мониторинга доступности ваших домашних сервисов.  
Она позволяет быстро увидеть, какие сервисы (Proxmox, AdGuard, Home Assistant, роутер и т.д.) работают, а какие нет, используя проверки по HTTP/HTTPS и ICMP (ping).

## ✨ Возможности

- 📡 **Двойная проверка** — HTTP/HTTPS и ICMP (ping) для каждого сервиса.
- 🗂️ **Группировка сервисов** — объединяйте сервисы в логические группы (например, «Сеть», «Умный дом»).
- 🔄 **Автоматическое обновление** — статусы обновляются каждые 5 секунд без перезагрузки страницы.
- 🎨 **Тёмная и светлая темы** — переключение одним кликом с сохранением выбора в `localStorage`.
- 📱 **Адаптивный дизайн** — удобно на компьютере, планшете и смартфоне.
- 🖱️ **Кликабельные карточки** — нажмите на карточку, чтобы открыть веб-интерфейс сервиса.
- 💡 **Подсказки при наведении** — на иконках HTTP/ICMP отображается полный адрес проверки.
- 🐳 **Готов к запуску в Docker** — лёгкая контейнеризация.

---

## 🛠️ Используемые технологии

- Backend: Python + Flask
- Мониторинг: requests (HTTP), ping3 (ICMP)
- Кеширование: cachetools (TTL-кеш для статусов)
- Параллелизм: concurrent.futures (быстрая проверка сервисов)
- Frontend: HTML5, CSS3 (адаптивная вёрстка)
- Иконки: Iconify (библиотека SVG-иконок)
- Темы: CSS-переменные, переключение через JavaScript
- Контейнеризация: Docker

## 📁 Конфигурация (services.conf)

Файл services.conf написан в формате JSON.
Поля:
- **name** — название сервиса (обязательно)
- **url** — адрес веб-интерфейса (если есть)
- **ip** — IP-адрес для ping (если есть)
- **icon** — идентификатор иконки из Iconify
- **category** — категория (отображается под названием)
- **verify_ssl** — false для самоподписанных сертификатов
